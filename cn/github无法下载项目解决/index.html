github 报错 Permission denied (publickey)

git clone 下载GitHub项目时产生报错

Description： 显示报错"git@github.com: Permission denied (publickey). Could not read from remote repository" 的解决方案

报错原因分析：

permission denied (publickey) 没有权限的的publickey，出现这种错误的原因可能：

$\Rightarrow$ 客户端与服务器未生成 ssh key

$\Rightarrow$ 客户端与服务器的 ssh_key 不匹配

所以我们需要在本地重新生成一次 ssh key ,服务器也要重新配置一次。

解决步骤：

客户端(本地) 生成 ssh key

在终端输入：

ssh-keygen -t rsa -C"youemail@example.com"

这个邮箱是你注册GitHub 账号时使用的邮箱。 回车;

​ 可以选择合适路径来保存 the key , 并两次确认输入 这个passpharse .

​ 这样我们完成了 ssh key 的生成工作。

​ 然后我们在终端输入：

ssh -v git@github.com

​ 最后会出下面两句

​ 再在终端输入

ssh-agent -s![gite2](/Users/stand/Desktop/Typora img/gite2.png)

接着在终端输入

ssh-agent -s

接着在执行: ssh-add ~/.ssh/id_rsa


id_rsa 被保存在了指定路径： Identity added: …这是ssh key文件路径的信息

配置服务端

打开你刚刚生成的id_rsa.pub，将里面的内容复制，进入你的github账号，在settings下，SSH and GPG keys下new SSH key，然后将id_rsa.pub里的内容复制到Key中，完成后Add SSH Key。

最后，验证Key

输入 ： ssh -T git@github.com

显示如下信息：

显示“Hi xxx! You’ve successfully authenticated, but GitHub does not provide shell access. ”，则说明我们的问题已经得到了解决！
